CONFIGURACIÓN DEL SISTEMA DE CONEXIÓN Y DAO (PHP + MYSQLI)
Este documento explica cómo funciona el sistema de base de datos del proyecto y cómo estructurar cualquier DAO para mantener el código ordenado y eficiente.

1. EL PUENTE: ARCHIVO 'conexion.php'
Este archivo contiene las credenciales de la base de datos y se encarga de abrir la tubería física con MySQL usando la extensión 'mysqli'.

Estructura del código:
PHP
<?php
class Conexion {
    private string $host = "localhost";
    private string $db_name = "tecnolost_db";
    private string $username = "root";
    private string $password = "";
    public $conn = null;

    public function conectar() {
        // mysqli_connect abre la conexión directa usando los datos de arriba
        $this->conn = mysqli_connect($this->host, $this->username, $this->password, $this->db_name);
        
        // Si hay un error en los datos (ej: base de datos mal escrita), salta este aviso
        if (!$this->conn) {
            echo "Error de conexión: " . mysqli_connect_error();
        }

        // Retorna la conexión activa para que la use el DAO
        return $this->conn;
    }
}
?>
2. EL CONTRATO: ARCHIVO 'dao.php'
Es una interfaz. Funciona como un molde o contrato obligatorio. Todo DAO que creemos tiene que tener, como mínimo, estas 5 funciones para que el sistema sea consistente.

Estructura del código:
PHP
<?php
interface dao 
{
    public function insertar($obj): void;
    public function actualizar($obj): void;
    public function eliminar($obj): void;
    public function buscarPorId(int $id);
    public function listar();
}
?>
NOTA: El ': void' al final de las primeras funciones le prohíbe a PHP retornar datos en esos métodos. Solo ejecutan la acción (insertar, borrar) y listo.

3. LA IMPLEMENTACIÓN: EJEMPLO 'admindao.php'
Acá es donde se junta todo de forma eficiente usando el CONSTRUCTOR.

¿Qué hace el constructor de forma automática?
Cuando en la lógica de la página hacemos un '$adminDao = new admindao();', automáticamente el constructor ejecuta 'new Conexion()' y llama a 'conectar()'. De esta manera, el objeto nace con la conexión ya abierta y guardada en la variable '$this->conexion'.

Estructura base para CUALQUIER archivo DAO:
PHP
<?php
include_once 'admin.php'; // Cambiar por el modelo que corresponda (ej: producto.php)
include_once 'dao.php';
include_once 'conexion.php';

class admindao implements dao {
    // Acá adentro se va a guardar la conexión activa que armó el constructor
    private $conexion; 

    // CONSTRUCTOR: Se ejecuta solo apenas se instancia la clase
    public function __construct() {
        $db = new Conexion();              // 1. Instancia el archivo de conexión
        $this->conexion = $db->conectar(); // 2. Llama a la función y deja la canilla abierta
    }

    // A partir de acá, TODAS las funciones usan '$this->conexion' directamente
    public function insertar($obj): void {
        $sql = "INSERT INTO administradores (nombre, email) VALUES (...)";
        mysqli_query($this->conexion, $sql); 
    }

    public function actualizar($obj): void {
        $sql = "UPDATE administradores SET ...";
        mysqli_query($this->conexion, $sql);
    }

    public function eliminar($obj): void {
        $sql = "DELETE FROM administradores WHERE id = ...";
        mysqli_query($this->conexion, $sql);
    }

    public function buscarPorId(int $id) {
        $sql = "SELECT * FROM administradores WHERE id = $id";
        $resultado = mysqli_query($this->conexion, $sql);
        return mysqli_fetch_assoc($resultado);
    }

    public function listar() {
        $sql = "SELECT * FROM administradores";
        $resultado = mysqli_query($this->conexion, $sql);
        return mysqli_fetch_all($resultado, MYSQLI_ASSOC);
    }

    // REGLA: Se pueden agregar más funciones extras acá abajo que no estén en la interfaz 
    // (Ejemplo: buscarPorEmail, validarLogin, etc.)
}
?>
